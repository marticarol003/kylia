# Histórico técnico (snapshots superados)

Documentos de la vista-resumen **anterior** del sistema, superados por la serie
`00`–`04` de `docs/tecnico/`. Se conservan por su detalle único que la serie
nueva aún no recoge — no por ser la referencia actual.

- **`arquitectura.md`** — diagrama Mermaid, decisiones de stack, **costes mensuales
  estimados** y **plan de disaster recovery**. Parcialmente desfasado (describe
  Supabase/Stripe como "próximamente"; hoy Supabase es central). Útil por el
  modelo de costes y el DR, que no están en la serie `0x-`.
- **`estado-y-roadmap.md`** — estado y roadmap a fecha 2026-05-19. Superado; el
  estado actual vive en `00-panorama-tecnico.md` y la memoria del proyecto.
